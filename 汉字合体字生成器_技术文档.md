# 汉字合体字生成器 - 产品技术文档

## 产品概述

一个网页应用，用户输入4个汉字，系统将其合成为一个艺术合体字图片。风格为意象化书法（如"一帆风顺"可合成为帆船形态的书法字）。

## 核心流程

```
用户输入4个汉字
    ↓
前端发送请求到后端
    ↓
后端调用 Gemini API（Nano Banana 2 模型）
  - 发送4个规则文档（拓扑编码库、几何优化库、风格特征库、语义校验库）作为上下文
  - 发送用户输入的4个字 + 生成指令
  - 调用2次，生成2张不同的合体字图片
    ↓
前端展示2张图片，用户选择喜欢的那张
    ↓
用户可下载/分享选中的图片
```

## 技术架构

### 前端：纯静态网页
- HTML + CSS + JavaScript
- 不需要框架（React/Vue都不需要，保持简单）
- 响应式设计，支持手机和电脑访问

### 后端：Node.js 服务
- Express 或类似的轻量框架
- 负责：接收前端请求 → 调用 Gemini API → 返回图片
- 为什么需要后端：API Key 不能暴露在前端

### API：Google Gemini - Nano Banana 2
- 模型标识：需要在 Google AI Studio 确认具体的 model name
- 功能：图像生成
- 输入：文本 prompt（包含规则文档内容 + 用户的4个字）
- 输出：生成的图片

## 页面设计

### 只有一个页面，包含3个状态：

#### 状态1：输入状态
- 页面顶部：产品名称 + 简短说明（如"把你的四字词语变成一个字"）
- 中间：4个输入框（每个框输入1个汉字）
- 输入校验：只允许输入中文汉字，每个框只允许1个字
- 下方：「生成」按钮
- 底部：可选的热门词语快捷入口（如"恭喜发财""一帆风顺""马到成功"等）

#### 状态2：生成中
- 显示加载动画
- 加载文案（如"正在创作中..."）
- 预计等待时间：10-30秒（取决于API响应速度）

#### 状态3：结果展示
- 并排展示2张生成的合体字图片
- 用户点击选择其中一张
- 选中后显示大图 + 操作按钮：
  - 「保存图片」：下载到本地
  - 「重新生成」：回到输入状态
  - 「换个词试试」：清空输入，回到输入状态

## 后端 API 设计

### POST /api/generate

**请求体：**
```json
{
  "characters": ["恭", "喜", "发", "财"]
}
```

**响应体：**
```json
{
  "success": true,
  "images": [
    "data:image/png;base64,xxxxx...",
    "data:image/png;base64,xxxxx..."
  ]
}
```

**错误响应：**
```json
{
  "success": false,
  "error": "生成失败，请重试"
}
```

### 后端处理逻辑

```
1. 接收前端传来的4个汉字
2. 拼接 prompt：
   - 系统指令（固定的）
   - 4个规则文档的内容（固定的，从服务器本地文件读取）
   - 用户输入的4个字（动态的）
3. 调用 Gemini API 2次（可以并行调用加快速度）
4. 拿到2张图片的 base64 数据
5. 返回给前端
```

## Gemini API 调用细节

### Prompt 拼接模板

```
你是一个专业的中国书法合体字设计师。你需要根据以下四个规则库，将用户给你的4个汉字合成为一个艺术合体字。

== 拓扑编码库 ==
{拓扑编码库.md 的完整内容}

== 几何优化库 ==
{几何优化库.md 的完整内容}

== 风格特征库 ==
{风格特征库.md 的完整内容}

== 语义校验库 ==
{语义校验库.md 的完整内容}

现在请将以下4个字合成为一个合体字：{用户输入的4个字，如"恭喜发财"}

要求：
1. 四个字的部件必须拆散重组，通过共享笔画融合为一个整体，不是简单地把四个完整的字摆在一起
2. 整体构图为正方形
3. 风格为传统毛笔书法，有墨迹质感和笔锋变化
4. 正面平视角度，90度正交俯视，无透视，无3D效果
5. 可以根据词语含义进行意象化设计（如将特征性的笔画异化为与词义相关的图形）
6. 请直接生成图片
```

## 注意事项

1. **API Key 安全**：绝对不能把 GEMINI_API_KEY 放在前端代码里，必须放在后端的 .env 文件中
2. **并发控制**：2次API调用可以用 Promise.all 并行执行，减少用户等待时间
3. **错误处理**：API 调用可能失败（超时、限流等），需要有重试逻辑和友好的错误提示
4. **图片大小**：生成的图片可能很大，考虑在后端做压缩再返回
5. **费用控制**：每次生成调用2次API，关注 Google AI 的计费方式，考虑是否需要加用户使用次数限制
6. **CORS**：后端需要配置 CORS 允许前端域名访问
