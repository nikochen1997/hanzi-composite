# 汉字合体字生成器 - 产品技术文档 v2

## 产品概述

一个网页应用，用户输入4个汉字，系统将其合成为一个艺术合体字图片。风格为意象化书法（如"一帆风顺"可合成为帆船形态的书法字）。

## 核心流程

```
用户输入4个汉字
    ↓
前端发送请求到后端
    ↓
后端调用 Gemini API（Nano Banana 2 模型）
  - 发送4个规则文档作为上下文
  - 发送用户输入的4个字 + 生成指令
  - 调用2次（并行），生成2张不同的合体字图片
    ↓
前端展示2张图片，用户选择喜欢的那张
    ↓
用户可下载/分享选中的图片
```

## 技术架构

### 前端：纯静态网页
- HTML + CSS + JavaScript（不需要React/Vue）
- 移动端优先的响应式设计
- 参考已有demo.html的设计风格（附在项目中）

### 后端：Node.js
- Express 轻量框架
- 接收前端请求 → 调用 Gemini API → 返回图片
- API Key 放在后端，不暴露给前端

### API：Google Gemini - Nano Banana 2
- 输入：文本 prompt（规则文档 + 用户的4个字）
- 输出：生成的图片

## 后端 API 设计

### POST /api/generate

**请求体：**
```json
{
  "characters": ["恭", "喜", "发", "财"]
}
```

**响应体（成功）：**
```json
{
  "success": true,
  "images": [
    "data:image/png;base64,xxxxx...",
    "data:image/png;base64,xxxxx..."
  ]
}
```

**响应体（失败）：**
```json
{
  "success": false,
  "error": "生成失败，请重试"
}
```

### 后端处理逻辑

```
1. 接收前端传来的4个汉字
2. 拼接prompt：
   - 系统指令（固定）
   - 4个规则文档内容（从服务器本地文件读取，固定）
   - 用户的4个字（动态）
3. 用 Promise.all 并行调用 Gemini API 2次
4. 返回2张图片的base64数据
```

```
== 几何优化库 ==
{几何优化库.md 的完整内容}

== 风格特征库 ==
{风格特征库.md 的完整内容}

== 语义校验库 ==
{语义校验库.md 的完整内容}

要求：
1. 四个字的部件必须拆散重组，通过共享笔画融合为一个整体，不是简单地把四个完整的字摆在一起
2. 整体构图为正方形

4. 正面平视角度，90度正交俯视，无透视，无3D效果
5. 可以根据词语含义进行意象化设计（如将特征性的笔画异化为与词义相关的图形）
6. 请直接生成图片
```

## 关键注意事项

1. **API Key 安全**：GEMINI_API_KEY 只能放在后端 .env 中
2. **并行调用**：2次API调用用 Promise.all 并行，减少等待时间
3. **错误处理**：API调用可能失败（超时、限流），需要重试逻辑和友好错误提示
4. **CORS**：后端配置 CORS 允许前端域名访问
5. **图片大小**：生成的图片可能很大，考虑后端压缩后再返回
6. **Gemini API具体调用方式**：开发前先去 Google AI Studio Playground 选 Nano Banana 2，点"Get code"获取准确的API调用代码
7. **历史记录**：用 localStorage 存储，参考demo.html的实现方式
